# Sprint 12 — wordsynk: Reconnection Config

## Context

After Sprint 11 cleaned d2a references from wordsynk, wordsynk still needs a lightweight connection point to the d2a pipeline — specifically so that when the pipeline targets the booking automation system, it knows where to find context.

Sprint 12 creates that connection point: a `d2a.config.json` that tells the pipeline how to reach and contextualize the wordsynk project.

## Steps

### Step 1: Create d2a.config.json

Write `d2a.config.json` in the wordsynk root with this content:

```json
{
  "endpoint": "http://localhost:3031",
  "targetRepo": "speeed76/cobalt-relay",
  "contextPath": "agents/worker/contexts/",
  "slackChannel": "C0AFCJT0XFH",
  "projectName": "wordsynk-automation",
  "description": "Booking automation system — Airtable calendar sync, scheduling, webhooks",
  "notes": "Context files in contextPath are used when d2a pipeline targets this repo"
}
```

Verify it's valid JSON:
```bash
python3 -m json.tool d2a.config.json && echo "JSON valid" || echo "JSON INVALID"
```

### Step 2: Ensure Context Directory Exists

The `contextPath` value points to `agents/worker/contexts/`. Create it if it doesn't exist:

```bash
mkdir -p agents/worker/contexts
ls agents/worker/contexts/ 2>/dev/null || echo "empty (that's fine)"
```

If there are already context files there, note them. If empty, that's expected — they're created by the pipeline.

### Step 3: Update CLAUDE.md Host Environment Section

Add a comment to the CLAUDE.md in the relevant section.

Find the section that mentions the host environment or external tools. Add:

```
**Forge context files**: `agents/worker/contexts/*.md` — generated by the d2a pipeline when targeting this repo. Read these if they exist; they describe the current scaffold context for the automation system.
```

If there's no existing section for this, add it to the end of CLAUDE.md in a section called "## External Integrations".

### Step 4: Verify

```bash
# JSON valid
python3 -m json.tool d2a.config.json
echo ""

# Contains required fields
python3 -c "
import json
with open('d2a.config.json') as f:
    d = json.load(f)
required = ['endpoint', 'targetRepo', 'contextPath', 'projectName']
for k in required:
    print(f'{k}: {d.get(k, \"MISSING\")}')
"
```

### Step 5: Commit

```bash
git add d2a.config.json CLAUDE.md
git status

git commit -m "config: add d2a.config.json for pipeline reconnection

Lightweight config file for when d2a pipeline targets this repo.
Points to mission-control endpoint :3031, cobalt-relay target,
and agents/worker/contexts/ for Forge-generated context files.

Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>"
```
